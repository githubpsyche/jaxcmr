Parameter Sensitivity

Explore how parameters affect model predictions
Note

For the full workflow, see the parameter shifting template notebook.

Overview

Understanding how CMR parameters affect model predictions is crucial for:

  • Interpreting fits: What does a high drift rate mean behaviorally?
  • Setting bounds: What parameter ranges produce realistic behavior?
  • Identifying parameters: Can different parameters produce similar effects?

Basic Parameter Sweep

Vary one parameter while holding others at baseline:

from jax import random
import jax.numpy as jnp
from jaxcmr.models.cmr import CMR
from jaxcmr.simulation import simulate_study_and_free_recall
from jaxcmr.analyses.spc import spc
import matplotlib.pyplot as plt

# Baseline parameters
base_params = {
    "encoding_drift_rate": 0.5,
    "start_drift_rate": 0.5,
    "recall_drift_rate": 0.5,
    "shared_support": 0.05,
    "item_support": 0.25,
    "learning_rate": 0.5,
    "primacy_scale": 1.0,
    "primacy_decay": 0.6,
    "choice_sensitivity": 0.6,
    "stop_probability_scale": 0.05,
    "stop_probability_growth": 0.2,
    "learn_after_context_update": True,
    "allow_repeated_recalls": False,
}

# Parameter values to test
drift_rates = [0.2, 0.4, 0.6, 0.8]

# Simulate for each value
list_length = 10
n_trials = 500
rng = random.PRNGKey(0)

results = {}
for drift in drift_rates:
    params = base_params.copy()
    params["encoding_drift_rate"] = drift

    all_recalls = []
    for i in range(n_trials):
        rng, rng_iter = random.split(rng)
        model = CMR(list_length, params)
        present = jnp.arange(1, list_length + 1)
        model, recalls = simulate_study_and_free_recall(model, present, rng_iter)
        all_recalls.append(recalls)

    results[drift] = jnp.stack(all_recalls)

Visualizing Effects

Plot how the serial position curve changes:

fig, ax = plt.subplots()

for drift, recalls in results.items():
    # Compute SPC
    curve = spc(recalls, list_length)
    ax.plot(range(1, list_length + 1), curve, label=f"drift={drift}")

ax.set_xlabel("Serial Position")
ax.set_ylabel("Recall Probability")
ax.legend()
ax.set_title("Effect of Encoding Drift Rate on SPC")
plt.show()

Common Parameter Effects

Encoding Drift Rate

Value Effect
Low (~0.2) Weak recency, more uniform recall
High (~0.8) Strong recency, temporal contiguity

Primacy Parameters

Change Effect
Higher primacy_scale Stronger primacy (better early item recall)
Higher primacy_decay Primacy effect drops off faster

Choice Sensitivity

Value Effect
Low (~0.2) More random recall order
High (~2.0) More deterministic (most active item wins)

Stop Parameters

Change Effect
Higher stop_probability_scale Shorter recall sequences
Higher stop_probability_growth Stopping increases faster with output position

Parameter Interactions

Some parameters interact—changing one may require adjusting another:

  • Drift rate + learning rate: Both affect association strength
  • Shared support + item support: Both affect baseline retrieval
  • Primacy scale + decay: Trade off in shaping the primacy curve

Practical Tips

  1. Start with published values: Use parameters from CMR papers as baselines
  2. Vary one at a time: Understand each parameter before varying multiple
  3. Check behavioral measures: Use SPC, CRP, and PNR to assess effects
  4. Consider identifiability: Some parameter combinations may produce identical behavior

## Basic Parameter Sweep

Vary one parameter while holding others at baseline:

```python
from jax import random
import jax.numpy as jnp
from jaxcmr.models.cmr import CMR
from jaxcmr.simulation import simulate_study_and_free_recall
from jaxcmr.analyses.spc import spc
import matplotlib.pyplot as plt

# Baseline parameters
base_params = {
    "encoding_drift_rate": 0.5,
    "start_drift_rate": 0.5,
    "recall_drift_rate": 0.5,
    "shared_support": 0.05,
    "item_support": 0.25,
    "learning_rate": 0.5,
    "primacy_scale": 1.0,
    "primacy_decay": 0.6,
    "choice_sensitivity": 0.6,
    "stop_probability_scale": 0.05,
    "stop_probability_growth": 0.2,
    "learn_after_context_update": True,
    "allow_repeated_recalls": False,

results = {}
```
