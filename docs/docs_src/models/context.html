Temporal Context

The drifting context representation in CMR

Temporal context is the core representation in CMR that makes memory “episodic”—it binds items to when they occurred, not just what they are.

What Is Temporal Context?

Temporal context is a vector that drifts as items are experienced. Think of it as a summary of recent mental activity that:

  1. Changes gradually — each new item shifts context, but doesn’t reset it
  2. Stays unit-length — normalization ensures consistent magnitude
  3. Preserves history — earlier items leave traces in current context

This produces a natural recency gradient: recent items share more context with the current state than distant items.

Mathematical Specification

Representation

Context is a unit-length vector of size \(N+1\) where \(N\) is the list length:

\[\mathbf{c} \in \mathbb{R}^{N+1}, \quad \|\mathbf{c}\| = 1\]

The extra dimension (index 0) represents the “start-of-list” context.

Initialization

At the start of the experiment:

\[\mathbf{c}_0 = [1, 0, 0, \ldots, 0]^T\]

Only the start-of-list unit is active. This state is preserved as initial_state to enable drift back at retrieval onset.

Context Drift (Integration)

When a new input \(\mathbf{c}^{IN}\) arrives with drift rate \(\beta\):

\[\mathbf{c}_{new} = \rho \mathbf{c}_{old} + \beta \mathbf{c}^{IN}\]

The scaling factor \(\rho\) ensures the result remains unit-length:

\[\rho = \sqrt{1 + \beta^2\left[(\mathbf{c}_{old} \cdot \mathbf{c}^{IN})^2 - 1\right]} - \beta(\mathbf{c}_{old} \cdot \mathbf{c}^{IN})\]

Input Normalization

Before integration, inputs are normalized:

\[\mathbf{c}^{IN} \leftarrow \frac{\mathbf{c}^{IN}}{\|\mathbf{c}^{IN}\|}\]

This ensures consistent drift behavior regardless of input magnitude.

Drift Rate Effects

The drift rate \(\beta\) controls how much context changes with each input:

\(\beta\) Value Effect
0.0 No change (context frozen)
0.3 Slow drift, high similarity between adjacent items
0.6 Moderate drift, balanced temporal precision
0.9 Fast drift, each item has distinct context
1.0 Complete replacement (only new input matters)

Lower drift rates create broader temporal “windows”—items studied close together share similar contexts. Higher rates create sharper distinctions.

Context During Study

During encoding, each item \(i\) produces a contextual input through the MFC memory:

\[\mathbf{c}^{IN}_i = M^{FC} \mathbf{f}_i\]

where \(\mathbf{f}_i\) is the item’s feature representation. Initially, this retrieves the item’s pre-experimental context unit. As associations form, it retrieves a blend of item-specific and experimentally-learned context.

The context then drifts:

\[\mathbf{c}_i = \rho_i \mathbf{c}_{i-1} + \beta_{enc} \mathbf{c}^{IN}_i\]

Context During Retrieval

Start of Retrieval

Before the first recall, context drifts toward the start-of-list state:

\[\mathbf{c}_{start} = \rho_{N+1} \mathbf{c}_N + \beta_{start} \mathbf{c}_0\]

This balances recency (from \(\mathbf{c}_N\)) with primacy (from \(\mathbf{c}_0\)):

  • High \(\beta_{start}\) → strong primacy, first items cued
  • Low \(\beta_{start}\) → strong recency, last items cued

After Each Recall

When item \(i\) is recalled, its context is reinstated:

\[\mathbf{c}^{IN} = M^{FC} \mathbf{f}_i\]

This retrieved context contains experimentally-learned associations—it resembles the context from when item \(i\) was studied. Context then drifts:

\[\mathbf{c}_{new} = \rho \mathbf{c}_{old} + \beta_{rec} \mathbf{c}^{IN}\]

This context reinstatement is the key to contiguity: items studied near item \(i\) had similar contexts, so they receive high activation when cued.

Implementation

from jaxcmr.components.context import TemporalContext, init

# Create context for a 16-item list
context = init(item_count=16)

# Context has 17 dimensions (16 items + start-of-list)
print(context.size)  # 17

# Initial state: only start-of-list active
print(context.state)  # [1, 0, 0, ..., 0]

# Integrate a new input
import jax.numpy as jnp
context_input = jnp.zeros(17).at[1].set(1.0)  # First item's context unit
new_context = context.integrate(context_input, drift_rate=0.5)

# State has shifted toward the input while staying unit-length
print(jnp.linalg.norm(new_context.state))  # 1.0

The TemporalContext Class

class TemporalContext(Pytree):
    """Drifting, unit-length context representation."""

    def __init__(self, item_count: int, size: int) -> None:
        self.size = size
        self.zeros = jnp.zeros(size)
        self.state = self.zeros.at[0].set(1)       # Current context
        self.initial_state = self.zeros.at[0].set(1)  # Start-of-list
        self.next_outlist_unit = item_count + 1

    def integrate(
        self,
        context_input: Array,
        drift_rate: float,
    ) -> "TemporalContext":
        """Returns context after integrating input."""
        context_input = normalize_magnitude(context_input)
        rho = jnp.sqrt(
            1 + jnp.square(drift_rate) *
            (jnp.square(self.state * context_input) - 1)
        ) - (drift_rate * (self.state * context_input))
        return self.replace(
            state=normalize_magnitude(
                (rho * self.state) + (drift_rate * context_input)
            )
        )

Expanded Context

For models needing out-of-list context (e.g., for repetition paradigms), use init_expanded:

from jaxcmr.components.context import TemporalContext

# Create context with extra capacity for out-of-list units
context = TemporalContext.init_expanded(item_count=16)
print(context.size)  # 33 (16 items + 16 outlist + 1 start)

# Get out-of-list input
outlist_input = context.outlist_input

# Integrate mixed in-list and out-of-list context
new_context = context.integrate_with_outlist(
    inlist_input=some_input,
    ratio=0.7,  # 70% in-list, 30% out-of-list
    drift_rate=0.5,
)

Why Unit-Length?

Maintaining unit-length context serves two purposes:

  1. Consistent retrieval strength — The dot product \(M^{CF} \mathbf{c}\) has bounded magnitude
  2. Interpretable similarity — \(\mathbf{c}_1 \cdot \mathbf{c}_2\) directly measures context similarity

Without normalization, activations would grow unboundedly as context accumulates history.

Visualizing Context Drift

The TemporalContext class includes visualization support in Jupyter:

context._repr_markdown_()  # Returns HTML img tag of heatmap

This shows the current context state as a heatmap, useful for debugging and understanding model behavior.
