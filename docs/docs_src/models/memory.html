Memory Systems

Linear vs Instance memory in CMR

CMR uses two memory systems that map between items and context. jaxcmr provides two implementations: linear associative memory (the default) and instance-based memory (MINERVA-2 style). These produce similar behavior in standard paradigms but differ in how they handle repeated items.

The Two Memory Systems

CMR requires two directional memories:

Memory Direction Function
MFC Item → Context Retrieves contextual input when an item is presented or recalled
MCF Context → Item Computes item activations given the current context

Both memories start with pre-experimental associations and acquire new associations during study.

Linear Associative Memory

The default implementation stores associations as a weight matrix.

Learning

When item \(\mathbf{f}\) and context \(\mathbf{c}\) are associated with learning rate \(\eta\):

\[\Delta W_{ij} = \eta \cdot f_i \cdot c_j\]

This is a Hebbian outer product—co-active units strengthen their connection.

Retrieval

Given a cue \(\mathbf{x}\), the output is a linear combination:

\[\mathbf{y} = W \cdot \mathbf{x}\]

Properties

  • Blends at storage: Multiple associations to the same output overlap in the weight matrix
  • Efficient: Retrieval is a single matrix-vector product
  • Lossy: Individual episodes cannot be perfectly separated

MFC Initialization

def init_mfc(list_length, parameters, context):
    # Pre-experimental: each item → unique context unit
    # Weight = 1 - learning_rate (room for new associations)
    return LinearMemory(
        jnp.eye(item_count, context_size, 1) * (1 - learning_rate)
    )

The offset of 1 skips the start-of-list unit (index 0), so item 1 maps to context unit 1.

MCF Initialization

def init_mcf(list_length, parameters, context):
    # Pre-experimental associations
    # shared_support: all context units → all items
    # item_support: each context unit → its item (extra)
    base = jnp.full((context_size - 1, item_count), shared_support)
    for i in range(item_count):
        base[i, i] = item_support
    # Start-of-list unit (index 0) has no associations
    return LinearMemory(jnp.vstack((zeros, base)))

Instance-Based Memory

An alternative implementation inspired by MINERVA 2 (Hintzman, 1984) stores each association as a separate trace.

Learning

When item \(\mathbf{f}\) and context \(\mathbf{c}\) are associated:

\[M_{trace_i} = [\mathbf{f}, \gamma \cdot \mathbf{c}]\]

Each study event creates a new row in the trace matrix. The learning rate \(\gamma\) scales the context portion.

Retrieval

Given a cue (partial probe), each trace is activated by its similarity:

\[A_i = (\text{cue} \cdot m_i)^\tau\]

The parameter \(\tau\) (trace selectivity) controls the sharpness of matching:

  • \(\tau = 1\): Linear similarity, gentle competition
  • \(\tau > 1\): Superlinear, strong matches dominate
  • \(\tau \to \infty\): Winner-take-all

The output is the weighted sum of traces:

\[\mathbf{echo} = \sum_i A_i \cdot m_i\]

Properties

  • Preserves episodes: Each association is stored separately
  • Flexible retrieval: \(\tau\) controls whether memories blend or compete
  • More traces = more computation: Scales with number of study events

Mathematical Comparison

Aspect Linear Instance
Storage Matrix \(W\) Trace matrix \(M\)
Learning \(W \leftarrow W + \eta \mathbf{f} \mathbf{c}^T\) \(M \leftarrow [M; \mathbf{f}, \gamma\mathbf{c}]\)
Retrieval \(W\mathbf{x}\) \(\sum_i (\mathbf{x} \cdot m_i)^\tau m_i\)
Episodes Blended Preserved
Parameters — \(\tau\) (trace selectivity)

When Does It Matter?

For single-presentation paradigms (each item studied once), both implementations produce similar behavior. Differences emerge with:

Repeated Items

When an item is presented twice:

  • Linear: Both presentations strengthen the same associations (additive)
  • Instance: Two separate traces exist; retrieval depends on which trace is activated

Instance memory allows selective retrieval of specific presentations, useful for: - Spacing effects - Source memory - Context-dependent recall of repetitions

Interference Paradigms

When new associations must override old ones:

  • Linear: Old associations persist (interference)
  • Instance: New traces can dominate with high \(\tau\)

Implementation

LinearMemory Class

class LinearMemory(Pytree):
    """Associative memory storing linear item–context links."""

    def __init__(self, state: Array):
        self.state = state  # Weight matrix

    def associate(self, in_pattern, out_pattern, learning_rate):
        """Add a new association."""
        return self.replace(
            state=self.state + learning_rate * jnp.outer(in_pattern, out_pattern)
        )

    def probe(self, in_pattern):
        """Retrieve output given input."""
        return jnp.dot(in_pattern, self.state)

InstanceMemory Class

class InstanceMemory(Pytree):
    """Instance-based memory model."""

    def __init__(self, state, probe, study_index, trace_activation_scale,
                 input_size, output_size):
        self.state = state  # Trace matrix
        self.trace_activation_scale = trace_activation_scale  # τ

    def associate(self, in_pattern, out_pattern, learning_rate):
        """Store a new trace."""
        trace = jnp.concatenate((in_pattern, out_pattern * learning_rate))
        return self.replace(
            state=self.state.at[self.study_index].set(trace),
            study_index=self.study_index + 1,
        )

    def probe(self, in_pattern):
        """Retrieve output via trace activation."""
        # Activate traces by similarity
        activation = jnp.dot(self.state, padded_input)
        activation = power_scale(activation, self.trace_activation_scale)
        # Sum weighted traces
        echo = jnp.dot(activation, self.state)
        return echo[output_portion]

Choosing a Memory System

Use Case Recommended
Standard free recall LinearMemory (simpler, faster)
Repetition paradigms InstanceMemory (preserves episodes)
Spacing effects InstanceMemory
Source memory InstanceMemory
Efficient fitting LinearMemory

Usage

Specify memory systems when creating a model:

from jaxcmr.models.cmr import CMR
import jaxcmr.components.linear_memory as LinearMemory
import jaxcmr.components.instance_memory as InstanceMemory

# Default: linear memory
model = CMR(
    list_length=16,
    parameters=params,
    mfc_create_fn=LinearMemory.init_mfc,
    mcf_create_fn=LinearMemory.init_mcf,
)

# Instance-based memory
params["mcf_trace_sensitivity"] = 3.0  # τ parameter
params["mfc_trace_sensitivity"] = 1.0

model = CMR(
    list_length=16,
    parameters=params,
    mfc_create_fn=InstanceMemory.init_mfc,
    mcf_create_fn=InstanceMemory.init_mcf,
)

Additional Parameters for Instance Memory

Parameter Description
mcf_trace_sensitivity \(\tau\) for context→item retrieval
mfc_trace_sensitivity \(\tau\) for item→context retrieval

Higher values increase selectivity; traces with high similarity dominate. Lower values blend more traces.

Theoretical Connections

The linear vs. instance distinction connects to broader debates in memory theory:

  • Connectionist models (e.g., back-propagation networks) use distributed representations like linear associative memory
  • Multiple-trace models (e.g., MINERVA, REM) store separate episodes like instance memory

For CMR, the choice affects how repetitions and interference are handled without changing the core temporal context mechanism.

References

  • Turner, B. M., et al. (2019). Approaching human performance in the generalized learning from feedback paradigm using linear and instance-based models. Computational Brain & Behavior, 2, 187-207.

## The Two Memory Systems

CMR requires two directional memories:

| Memory | Direction | Function |
|--------|-----------|----------|
| MFC | Item → Context | Retrieves contextual input when an item is presented or recalled |
| MCF | Context → Item | Computes item activations given the current context |

Both memories start with pre-experimental associations and acquire new associations during study.

## Instance-Based Memory

An alternative implementation inspired by MINERVA 2 (Hintzman, 1984) stores each association as a separate trace.

### Learning

When item $\mathbf{f}$ and context $\mathbf{c}$ are associated:

$$M_{trace_i} = [\mathbf{f}, \gamma \cdot \mathbf{c}]$$

Each study event creates a new row in the trace matrix. The learning rate $\gamma$ scales the context portion.

### Retrieval

Given a cue (partial probe), each trace is activated by its similarity:

$$A_i = (\text{cue} \cdot m_i)^\tau$$

The parameter $\tau$ (trace selectivity) controls the sharpness of matching:

- $\tau = 1$: Linear similarity, gentle competition
- $\tau > 1$: Superlinear, strong matches dominate
- $\tau \to \infty$: Winner-take-all

The output is the weighted sum of traces:

$$\mathbf{echo} = \sum_i A_i \cdot m_i$$

### Properties

- **Preserves episodes**: Each association is stored separately
- **Flexible retrieval**: $\tau$ controls whether memories blend or compete
- **More traces = more computation**: Scales with number of study events

### Mathematical Comparison

| Aspect | Linear | Instance |
|--------|--------|----------|
| Storage | Matrix $W$ | Trace matrix $M$ |
| Learning | $W \leftarrow W + \eta \mathbf{f} \mathbf{c}^T$ | $M \leftarrow [M; \mathbf{f}, \gamma\mathbf{c}]$ |
| Retrieval | $W\mathbf{x}$ | $\sum_i (\mathbf{x} \cdot m_i)^\tau m_i$ |
| Episodes | Blended | Preserved |
| Parameters | — | $\tau$ (trace selectivity) |

## When Does It Matter?

For single-presentation paradigms (each item studied once), both implementations produce similar behavior. Differences emerge with:

### Repeated Items

When an item is presented twice:

- **Linear**: Both presentations strengthen the same associations (additive)
- **Instance**: Two separate traces exist; retrieval depends on which trace is activated

Instance memory allows selective retrieval of specific presentations, useful for:
- Spacing effects
- Source memory
- Context-dependent recall of repetitions

### Interference Paradigms

When new associations must override old ones:

- **Linear**: Old associations persist (interference)
- **Instance**: New traces can dominate with high $\tau$

    """Associative memory storing linear item–context links."""

## Choosing a Memory System

| Use Case | Recommended |
|----------|-------------|
| Standard free recall | LinearMemory (simpler, faster) |
| Repetition paradigms | InstanceMemory (preserves episodes) |
| Spacing effects | InstanceMemory |
| Source memory | InstanceMemory |
| Efficient fitting | LinearMemory |
