This document records our progress and key decisions as we build a brain-based version of eCMR using EEG data from Zarubin et al. (2020).

Right now, we are making a simple prototype to demonstrate item-level LPP signals (an EEG measure) can improve a retrieved context model’s predictions of the emotion boost to memory.

Later, we may try other model designs, fitting methods, and ways to evaluate the results.

1 Data Preparation

The primary table comes from Single_Trial_Behavioural_and_EEG_Data_Z.csv located at Z:\Talmi lab space\Archive\Secondary_Data_Analysis_Zarubin_From_Robin\Manuscript\files from robin nov24\.

This was missing some study events and their recall results. So I merged it with the behavioral log All_Included_Subjects.csv located at Z:\Talmi lab space\Archive\Secondary_Data_Analysis_Zarubin_From_Robin\Data\Behaviour\Behaviour_csv_files\. This file has recall data for every non-buffer item but no LPP scores. Before merging, I confirmed no data present in the EEG file disagreed with the behavioral log. For example, no items marked as recalled in the EEG file were marked as not recalled in the behavioral log.

The merged frame contains 6,840 study events (342 participant × list trials, 20 items per list after trimming the two primacy buffer items). Among these, 371 study events (5.42 %) lacked EEG measurements. The gaps affected both valence conditions: negative lists contributed 125 of 2,266 events (5.52 %; mean 0.37 ± 0.65 items per trial), whereas neutral lists contributed 246 of 4,574 events (5.38 %; mean 0.72 ± 0.93 items per trial). Overall, 202 of 342 lists (59.1 %) needed at least one interpolated observation, with 1.84 +/- 1.01 missing items among impacted trials (median 2; maximum 6). Subject-level exposure ranged from 2 to 19 items (3.9 to 10 % of each participant’s 180 study events), so the imputation step is methodologically substantive.

LPP amplitudes were interpolated hierarchically. For study events still missing LPP scores, I filled values using each person’s average for items of the same emotion in the same trial.

Each trial originally had 22 items. The first two were removed during preprocessing to reduce primacy effects and cannot be retrieved. This is not ideal for CMR-style models, which often use early items to shape context. For now, we will just ignore the missing items, treating each list as having 20 items.

The dataset does not include recall order. It only marks whether each item was recalled. Because of this, even early prototypes require specifying custom loss functions for model fitting that do not rely on recall order information.

3 Loss Functions For Model Fitting

We face two main challenges when defining loss functions for this dataset. First, models like CMR and eCMR predict the order of recalls, but our data only say whether each item was recalled. This forces a simpler question: does the set of recalled items match what the model predicts?

Second, the first two items of each list were removed from the data. This helps reduce primacy effects for emotion analyses, but it removes useful information for modeling how lists were encoded and retrieved.

Together, these issues prevent standard likelihood tests on recall sequences. For now, we ignore the missing items and fit models to predict the set of recalled and unrecalled items that we do observe. We also skip modeling how many items were recalled in each trial. CMR and eCMR do not make strong counts predictions without extra rules for stopping recall. Dropping recall counts keeps the task focused on which items were recalled rather than how many or in what order.

3.1 Set Likelihood

The best way to evaluate models is to compute the likelihood of the observed data under each model. We normally do this by comparing predicted and observed recall sequences. But our data lack recall order information.

Without recall order, we must rate models by the probability they give to the set of recalled items. Many recall orders can lead to the same set. A complete set likelihood function would thus compute and sum the probabilities of all recall sequences that produce the observed set. This is possible in theory, but it is very costly in practice. The number of possible recall sequences grows factorially with the number of recalled items. Summing over all sequences quickly becomes infeasible.

A Monte-Carlo approach sidesteps this problem by evaluating only a sample of possible recall sequences. But there are many ways to implement this idea. The approach I first considered focused sampling on simulations from the model. For each subject and trial, we can simulate many recall sequences. We can then check how many simulated sequences match the observed set of recalled items. The proportion of matching sequences gives an unbiased estimate of the set likelihood given the model and parameter set.

The problem with this approach is that it often gives zero likelihood to reasonable parameter sets. If no simulated sequence matches the observed set exactly, the estimated likelihood is zero. This is especially likely when many items are recalled, since the number of possible recall sequences grows quickly with the number of recalled items. Even when some simulated sequences match the observed set, the estimate can be noisy unless many simulations are run per subject and trial. This makes fitting costly and slow.

I’m now exploring a cleaner approach. Instead of sampling model simulations of a given trial, we can sample permutations of the observed set of recalled items. Each permutation is a possible recall sequence that produces the observed set. For each permutation, we can compute its likelihood under the model. Aggregating these likelihoods gives an estimate of the set likelihood.

This approach avoids zero likelihoods for reasonable parameter sets, since every permutation of the observed set has some non-zero likelihood under the model. It also supports Monte-Carlo estimation to control cost, since we can choose how many permutations to sample per subject and trial. Finally, it preserves the advantages of likelihood-based model fitting and comparison over alternatives like mean squared error (MSE). Once initial prototyping is done, I’ll implement this approach for model fitting and evaluation.

3.2 Mean Squared Error (MSE) Over Summary Statistics

Given initial challenges implementing a set-likelihood-based fitting procedure, we are starting with a simpler approach for prototyping. We instead use mean squared error between a chosen summary statistic in the data and the same statistic from simulations. This is common in retrieved-context model evaluation and fits our constraints.

The procedure is simple. We choose a summary statistic from the experiment. We simulate the model with a parameter set. We compute the same statistic on the simulations. We then compute the mean squared difference between the two numbers.

This score tells us how well a parameter set for the model reproduces the pattern we care about. If the best score is still poor, the model is likely not adequate. Fitting by MSE does not find the maximum-likelihood parameters and loses trial-level detail, but it is practical, scalable, and clear.

We currently use the category serial position curve for model fitting and evaluation. It computes recall by position separately for negative and neutral items. This tests both position effects and the emotion boost at the same time and is a better target for neurally informed eCMR.

The MSE can compose over multiple statistics if needed. We may add a neural-behavioral link statistic later to ensure the model captures LPP-recall relationships. For now, we keep it simple. And we expect to switch back to likelihood-based fitting anyway once basic prototypes work.

4 Modeling

We start with standard CMR, which ignores emotion and LPP. This baseline checks that the emotion boost to memory does not appear without additional mechanisms.

Next, we build and select between simple versions of eCMR that can capture the emotion-enhanced memory effect. Our candidate models focus on the proposal that the boosted recall for emotional items over neutral items comes from extra attention during study, boosting the strength of context-to-item associations for emotional items relative to neutral items. We set aside other possible mechanisms of the emotion-enhanced memory effect for now, such as boosted integration rates for emotional or temporal context, or enhanced item-to-context learning for emotional items. Many model variants are still possible within this attention-focused framework. We explore some issues below.

To preserve the interpretability of downstream extensions and avoid overfitting, we focus on simple eCMR variants that use as few extra parts as necessary. We only add what is needed to capture available behavioral benchmarks, especially the emotion-enhanced memory effect. We also check that these models cannot capture the link between LPP and recall without directly using LPP data.

Once a simple eCMR specification is chosen, we add a neural extension that uses item-level LPP to scale the strength of emotional attention. This model has two goals. It should beat both baseline CMR and the simple eCMR on the emotion effect. It should also capture how LPP relates to recall for both emotional and neutral items. Meeting both goals will give us a solid starting model that we can improve later with better data and stronger techniques for model fitting and evaluation.

The remainder of this section is structured around modeling decisions. Each subsection describes a dilemma we faced, the choice we made, the reasoning behind it, and results from testing it.

4.1 Single vs Dual Context Representations

Full eCMR adds a separate emotional context and memories that link emotion to items and back to context. In that setup, attention mainly raises the learning rates in those emotion links. At recall, reinstating the emotional context then favors emotional items.

A simpler specification may be enough to capture the emotion-enhanced memory effect in our data. This specification matches baseline CMR and has only one context (temporal). This version skips emotional context and memories linking emotion to items and back to context. An emotional-attention parameter directly scales the strength of temporal context-to-item links during study. Just one extra parameter beyond baseline CMR controls this scaling. We call this version pure-attention eCMR.

An alternative simplification called two-context eCMR keeps two contexts but but fixes the emotional context to fully track the most recent item. That removes the need to learn emotion feature to context links or configure an emotional context integration rate. Emotional context-to-feature memory is maintained with a emotional-attention parameter that sets how strongly emotional context cues emotional items. Thus, like pure-attention eCMR, this version only adds one parameter beyond baseline CMR.

Pure-attention eCMR says attention directly strengthens context-to-item links. The two-context version says emotional items help reinstate emotional context, which then favors other emotional items. Both models thus propose that attention during study drives the emotion effect, but the two-context version explicitly proposes that emotional items cluster together at recall via emotional context reinstatement. We cannot compare these proposals directly because the present dataset lacks recall order. Even so, this model can still make different predictions about the composition of recalled items for lists with mixed emotional and neutral items. We can at least compare their ability to capture the emotion-enhanced memory effect.

4.1.1 Parameter Sensitivity Tests

These results show that scaling up the parameter increases recall rates for negative items while reducing recall rates for neutral items, building confidence that the parameter can improve the baseline model’s ability to address the emotion-enhanced memory effect. Both model variants show similar sensitivity to changes in the emotional-attention scaling parameter, despite their different mechanisms. This suggests that either model could potentially capture the emotion-enhanced memory effect with appropriate parameter settings.

4.2 Additive vs Multiplicative Interaction Between Primacy and Emotional Attention

The specifications for eCMR and CMR3 both seem to configure a multiplicative relationship between emotional-attention and primacy-attention. That means their effects combine by multiplying rather than adding. If primacy attention is high at the start of the list and the emotion scale is also high, the total boost for an early emotional item is large. For a late item where primacy is low, the same emotion scale gives a much smaller boost. This predicts a larger gap between emotional and neutral recall early in the list than later.

Here is an example. Suppose primacy at position 1 equals 10 and the emotion scale equals 3. The boost for an emotional item is 10 times 3 which is 30. The boost for a neutral item is 10. The gap is 20. At a later position where primacy equals 1, the emotional boost is 3 and the neutral boost is 1. The gap is 2. The early gap is ten times larger.

Our data and current pipeline give only a limited test of this question because we lack recall order and recall data for the first two list items in each trial. Still, we can still compare additive and multiplicative fits to recall rates by position and category. Initial tests apply an MSE-based fit to a simple extension of baseline CMR where emotional items get either a multiplicative or additive boost to context-to-item associations during encoding (pure-attention eCMR).

We find that the additive variant better captures the emotion-enhanced memory effect without over-predicting recall rates for negative items in early study positions. This question can be revisited later with better data, more powerful fitting methods, and more complex models. For now, we assume an additive relationship between primacy and emotional attention.

Since two-context eCMR has two context-to-item memory structures (temporal and emotional), it can implement additive, multiplicative, or no relationship between primacy and emotional attention in each memory structure. If we assume that emotional attention is always applied in emotional-context-to-item learning and primacy is always applied in temporal-context-to-item learning, this specifies 3 x 3 = 9 possible relationships. A couple of these have been evaluated, all finding that keeping emotional attention and primacy independent works best. Thus, for two-context eCMR we also assume an additive/independent relationship between primacy and emotional attention.

4.3 Neurally-Informed Extension

We want our neurally-informed eCMR to use item-level Early LPP data to both:

  1. Improve model fits to the emotion-enhanced memory effect; and
  2. Capture how LPP relates to recall within each item category.

To constrain the design of this extension, we lean on the linearity assumptions embodied in earlier generalized mixed effects model results. The model predicted recall from item emotionality and single trial Early LPP amplitude, finding strong effects of both predictors and their interaction. Recalled negative items had higher Early LPP than unrecalled negative items, while neutral items showed little difference. From the GLMM results, we make two working assumptions. First, LPP affects recall for emotional items in a roughly linear way. Second, the context to feature learning rate configures recall rate.

We thus design a simple neurally-informed eCMR extension that uses Early LPP to scale the emotional-attention parameter for emotional items only. Neutral items remain unaffected. We use a lpp_slope parameter to set how strongly Early LPP spreads recall rates apart for emotional items. An lpp_threshold parameter replaces emotion_scale from earlier variants and sets the cutoff between recalled and unrecalled emotional items. Depending on whether Early LPP is above or below this threshold, emotional attention increases or decreases relative to the neutral baseline.

This slope and threshold approach gives a clear, data grounded starting point for testing approaches for relating LPP to memory in eCMR. After it works, we can explore nonlinear features if we want.

5 Discussion and Next Steps

Our goals for this stage of our modeling were twofold. First, we wanted to build a simple eCMR variant that could capture the emotion-enhanced memory effect in our data. Second, we wanted to extend that model to use item-level Early LPP data to both improve fits to the emotion effect and capture how LPP relates to recall within each item category.

In this stage of the project, we focused on two benchmark statistics: the category-split serial position curve and the relationship between Early LPP and recall for negative items. We used mean squared error between simulated and observed statistics as our loss function for model fitting and evaluation. We compared baseline CMR, two simple eCMR variants (pure-attention and two-context), and neurally-informed extensions of those eCMR variants. We found that both eCMR variants but not baseline CMR could capture the emotion-enhanced memory effect to a reasonably high degree. The neurally-informed extensions primarily additionally gained the ability to capture the relationship between Early LPP and recall for negative items, with small or no improvements to fits to the emotion effect. We thus partially met both our goals for this stage of the project.

Our most ambiguous result is that none of the presented eCMR variants, not even neurally-informed extensions, clearly outperformed the other when fitting the behavioral emotion-enhanced memory effect. This outcome probably reflects three realities:

  • Limited information in LPP signal. Early LPP data captures differences in recall rate for emotional items. However, the category-split serial position curve measures differences in recall rate between emotional and neutral items. Thus, Early LPP data may not directly inform fits to that statistic. It is best suited for configuring predictions about which emotional items are most likely to be recalled, rather than how emotional items differ from neutral items overall.
  • Lack of theoretical constraint in order-free data and summary statistics. The primary difference between pure-attention and two-context eCMR is in their predictions about how emotional items cluster together in recall sequences. Two-layer eCMR predicts that boosted recall for emotional items is contingent on reinstating emotional context that cues other emotional items. Pure-attention eCMR predicts that emotional items are recalled more often regardless of whether the last recalled item was emotional or neutral. Without recall order data, we cannot test this key theoretical difference. Both models can make similar predictions about overall recall rates for emotional and neutral items, making it hard to distinguish them.
  • Informal model fitting and evaluation procedure. Our current fitting procedure uses mean squared error between simulated and observed summary statistics. We primarily compare models by visually inspecting fits to those statistics. Even if we directly compared final MSE scores, differences in the number of parameters per model and the lack of formal likelihoods would hamper strong conclusions.

We aren’t necessarily interested in selecting between possible eCMR specifications with this project. Instead, our goal is to point the way toward neurally-informed retrieved-context models of emotional memory. But as is, we haven’t clearly shown that adding LPP data to eCMR helps capture behavioral patterns of recall beyond what eCMR can already do.

While some of these problems cannot be overcome with the present dataset, others can probably be resolved by shifting the focus of model fitting and evaluation. We can directly optimize and test models for their ability to predict (give higher likelihoods) to the specific items recalled or not recalled in each trial. This shift would better align our modeling goals with the signal we understand LPP to provide. It would also better distinguish between eCMR variants by comparing their ability to predict trial-level data rather than summary statistics and enabling formal comparison tests that account for differences in model complexity.

We know that LPP relates to recall at the item level, and that pure-attention and two-context eCMR variants give different probability distributions over possible recall sets of mixed emotional and neutral item lists. But our current benchmarks do not directly test these relationships. Moving to a trial-level, likelihood-based approach for model fitting and evaluation should help clarify whether adding LPP data to eCMR improves its ability to capture emotional memory effects, and whether this dataset is useful for distinguishing between eCMR variants more generally.

So here are the remaining steps as I see them:

  1. Swap to a set-likelihood-based fitting and evaluation procedure as described earlier. Redo model fitting with this new procedure and simulate summary statistics with the best-fitting parameters.
  2. Compare model fits to both summary statistics and trial-level set likelihoods. Check whether adding LPP data improves eCMR’s ability to capture the emotion-enhanced memory effect and whether trial likelihoods help distinguish between eCMR variants after accounting for model complexity.
  3. Optional: Refocus evaluation on the complete eCMR specification, contrasting it against a neurally-informed extension and against baseline CMR. Unnecessary if goal is just to prove a neurally-informed baseline.
  4. Optional: Refine bootstrapped confidence intervals for summary statistics to better capture uncertainty in model predictions. Unnecessary since current approach already matches standard practice and other results (e.g., GMM outcomes) already confirm existing patterns.
  5. Write up results for internal review and eventual publication, Methods-focused talk, grant materials, etc.
