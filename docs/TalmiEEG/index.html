This document records our progress and key decisions as we build a brain-based version of eCMR using EEG data from Zarubin et al. (2020).

Right now, we are making a simple prototype to test if item-level LPP signals (an EEG measure) can improve eCMR’s predictions of the emotion boost to memory.

Later, we may try other model designs, fitting methods, and ways to evaluate the results.

1 Data Preparation

I began with Single_Trial_Behavioural_and_EEG_Data_Z.csv, which was missing some study events and their recall results. I merged it with All_Included_Subjects.csv, which has all study events but no Early or Late LPP scores.

For study events still missing LPP, I filled values using each person’s average for items of the same emotion in the same trial. I then checked the final dataset. It has 9 trials per participant, 20 study events per trial, and a recall outcome for every event.

Each trial originally had 22 items. The first two were removed during preprocessing to reduce primacy effects and cannot be retrieved. This is not ideal for CMR-style models, which often use early items to shape context, but we will work around it.

The dataset does not include recall order. It only marks whether each item was recalled. Because of this, even early prototypes need custom loss functions for model fitting.

The pattern is clearer for Early LPP than for Late LPP, which supports focusing on Early LPP in the model. The pattern also suggests that we only use LPP to modulate recall for NEGATIVE items, leaving NEUTRAL items unaffected.

3 Loss Functions For Model Fitting

We face two main challenges when defining loss functions for this dataset. First, models like CMR and eCMR predict the order of recalls, but our data only say whether each item was recalled. This forces a simpler question: does the set of recalled items match what the model predicts?

Second, the first two items of each list were removed from the data. This helps reduce primacy effects for emotion analyses, but it removes useful information for modeling how lists were encoded and retrieved.

Together, these issues prevent standard likelihood tests on recall sequences. For now, we ignore the missing items and fit models to predict the set of recalled and unrecalled items that we do observe. We also skip modeling how many items were recalled in each trial. CMR and eCMR do not make strong counts predictions without extra rules for stopping recall. Dropping recall counts keeps the task focused on which items were recalled rather than how many or in what order.

3.1 Set Likelihood

The best way to evaluate models is to compute the likelihood of the observed data under each model. We normally do this by comparing predicted and observed recall sequences. But our data lack recall order information.

Without recall order, we must rate models by the probability they give to the set of recalled items. Many recall orders can lead to the same set. The exact solution would sum the probabilities of all orders that match the set. That grows too fast to compute for real data.

A practical fix is Monte Carlo sampling. For each trial and subject, we repeatedly simulate the model under candidate parameters and track the proportion of simulated recalls that match the observed set. This gives an unbiased estimate of set likelihood, but it adds sampling noise and can be costly. Because reliable results need many samples per trial, we leave a full test of this method for later stages.

3.2 Mean Squared Error (MSE) Over Summary Statistics

Given these limits, we use mean squared error between a chosen summary statistic in the data and the same statistic from simulations. This is common in retrieved-context model evaluation and fits our constraints.

The procedure is simple. We choose a summary statistic from the experiment. We simulate the model with a parameter set. We compute the same statistic on the simulations. We then compute the mean squared difference between the two numbers.

This score tells us how well a parameter set for the model reproduces the pattern we care about. If the best score is still poor, the model is likely not adequate. Fitting by MSE does not find the maximum-likelihood parameters and loses trial-level detail, but it is practical, scalable, and clear.

We currently use the category serial position curve for model fitting and evaluation. It computes recall by position separately for negative and neutral items. This tests both position effects and the emotion boost at the same time and is a better target for neurally informed eCMR.

The MSE can compose over multiple statistics if needed. We may add a neural-behavioral link statistic later to ensure the model captures LPP-recall relationships. For now, we keep it simple.

4 Modeling

We start with standard CMR, which ignores emotion and LPP. This baseline checks that the emotion boost to memory does not appear without additional mechanisms.

Next, we build and select between simple versions of eCMR that can capture the emotion-enhanced memory effect. Our candidate models focus on the proposal that the boosted recall for emotional items over neutral items comes from extra attention during study, boosting the strength of context-to-item associations for emotional items relative to neutral items. We set aside other possible mechanisms of the emotion-enhanced memory effect for now, such as boosted integration rates for emotional or temporal context, or enhanced item-to-context learning for emotional items. Many model variants are still possible within this attention-focused framework. We explore some issues below.

To preserve the interpretability of downstream extensions and avoid overfitting, we focus on simple eCMR variants that use as few extra parts as necessary. We only add what is needed to capture available behavioral benchmarks, especially the emotion-enhanced memory effect. We also check that these models cannot capture the link between LPP and recall without directly using LPP data.

Once a simple eCMR specification is chosen, we add a neural extension that uses item-level LPP to scale the strength of emotional attention. This model has two goals. It should beat both baseline CMR and the simple eCMR on the emotion effect. It should also capture how LPP relates to recall for both emotional and neutral items. Meeting both goals will give us a solid starting model that we can improve later with better data and stronger techniques for model fitting and evaluation.

The remainder of this section is structured around modeling decisions. Each subsection describes a dilemma we faced, the choice we made, the reasoning behind it, and results from testing it.

4.1 Single vs Dual Context Representations

Full eCMR adds a separate emotional context and memories that link emotion to items and back to context. In that setup, attention mainly raises the learning rates in those emotion links. At recall, reinstating the emotional context then favors emotional items.

A simpler specification may be enough to capture the emotion-enhanced memory effect in our data. This specification matches baseline CMR and has only one context (temporal). This version skips emotional context and memories linking emotion to items and back to context. An emotional-attention parameter directly scales the strength of temporal context-to-item links during study. Just one extra parameter beyond baseline CMR controls this scaling. We call this version pure-attention eCMR.

An alternative simplification called two-context eCMR keeps two contexts but but fixes the emotional context to fully track the most recent item. That removes the need to learn emotion feature to context links or configure an emotional context integration rate. Emotional context-to-feature memory is maintained with a emotional-attention parameter that sets how strongly emotional context cues emotional items. Thus, like pure-attention eCMR, this version only adds one parameter beyond baseline CMR.

Pure-attention eCMR says attention directly strengthens context-to-item links. The two-context version says emotional items help reinstate emotional context, which then favors other emotional items. Both models thus propose that attention during study drives the emotion effect, but the two-context version explicitly proposes that emotional items cluster together at recall via emotional context reinstatement. We cannot compare these proposals directly because the present dataset lacks recall order. Even so, this model can still make different predictions about overall recall for emotional and neutral items. We can at least compare their ability to capture the emotion-enhanced memory effect.

4.1.1 Parameter Sensitivity Tests

These results show that scaling up the parameter increases recall rates for negative items while reducing recall rates for neutral items, building confidence that the parameter can improve the baseline model’s ability to address the emotion-enhanced memory effect. Compared to pure-attention eCMR, the two-context eCMR variant shows a similar sensitivity to changes in the emotional-attention scaling parameter, with increases in the parameter boosting recall rates for negative items while reducing recall rates for neutral items. The Two-Context eCMR variant appears to produce a bigger boost to recall rates for the first and last study positions for negative items, while the pure-attention variant seems to apply some kind of soft ceiling on recall rates for negative items in the first and last study positions.

4.1.3 Discussion

Neither model makes substantially better predictions of the emotion-enhanced memory effect as measured by category-SPC than the other. Both models struggle to capture the emotion-enhanced memory effect adequately. Most critically, they underestimate recall rates for negative items past study position 10. They also overestimate recall rates for neutral items, particularly in early study positions between 3 and 8. We hope that adding neural data will help fix these issues. For now, we default to using pure-attention eCMR as our base model because of its relative simplicity. If initial neural extensions do not help, we can revisit this choice later. (In practice, both models are usually evaluated, but reports in this document focus on pure-attention eCMR unless the two-context variant shows a different pattern of results.)

4.2 Additive vs Multiplicative Interaction Between Primacy and Emotional Attention

The specifications for eCMR and CMR3 both seem to configure a multiplicative relationship between emotional-attention and primacy-attention. That means their effects combine by multiplying rather than adding. If primacy attention is high at the start of the list and the emotion scale is also high, the total boost for an early emotional item is large. For a late item where primacy is low, the same emotion scale gives a much smaller boost. This predicts a larger gap between emotional and neutral recall early in the list than later.

Here is an example. Suppose primacy at position 1 equals 10 and the emotion scale equals 3. The boost for an emotional item is 10 times 3 which is 30. The boost for a neutral item is 10. The gap is 20. At a later position where primacy equals 1, the emotional boost is 3 and the neutral boost is 1. The gap is 2. The early gap is ten times larger.

Our data and current pipeline give only a limited test of this question because we lack recall order and recall data for the first two list items in each trial. Still, we can still compare additive and multiplicative fits to recall rates by position and category. Initial tests apply an MSE-based fit to a simple extension of baseline CMR where emotional items get either a multiplicative or additive boost to context-to-item associations during encoding (pure-attention eCMR).

We find that the additive variant better captures the emotion-enhanced memory effect without over-predicting recall rates for negative items in early study positions. This question can be revisited later with better data, more powerful fitting methods, and more complex models. For now, we assume an additive relationship between primacy and emotional attention.

Since two-context eCMR has two context-to-item memory structures (temporal and emotional), it can implement additive, multiplicative, or no relationship between primacy and emotional attention in each memory structure. If we assume that emotional attention is always applied in emotional-context-to-item learning and primacy is always applied in temporal-context-to-item learning, this specifies 3 x 3 = 9 possible relationships. A couple of these have been evaluated, all finding that keeping emotional attention and primacy independent works best. Thus, for two-context eCMR we also assume an additive/independent relationship between primacy and emotional attention.

4.3 Neurally-Informed Extension

We now turn to exploring how a neural signal that hints at whether negative items will be recalled can lift model performance. Our goal is to show that the current data can support such an improvement, leaving the search for the best long term solution for later work.

We ground this prototype in earlier results. A generalized mixed effects model predicted recall from item emotionality and single trial Early LPP amplitude:

Memory ~ 1 + Condition * EarlyLPP + (1 + Condition * EarlyLPP | Participant)

Tests showed strong effects of Condition, Early LPP, and their interaction. Recalled negative items had higher Early LPP than unrecalled negative items. Neutral items showed little difference. Our plots of LPP by recall status match this pattern.

These results suggest an eCMR extension that changes learning for emotional items only, using Early LPP as the driver and leaving neutral items unchanged. If Early LPP separates recalled from unrecalled negative items, the model may ideally reduce learning for negative items with low LPP, even below the neutral baseline.

From the GLMM results, we make two working assumptions. First, LPP affects recall for emotional items in a roughly linear way. Second, the context to feature learning rate configures recall rate. To configure learning rates for emotional items based on LPP, we thus use a simple emod = m*x + b linear function.

Here x is the LPP value, m is the parameter lpp_slope, b is the parameter lpp_threshold which replaces emotion_scale, and emod is the adjusted learning rate for emotional items. In a good fit, the cutoff implied by lpp_threshold falls between unrecalled and recalled negative items. Items below the cutoff get a negative emod and items above it get a positive emod. On the other hand, lpp_slope sets how strongly LPP spreads their recall rates apart.

This slope and threshold approach gives a clear, data grounded starting point for testing approaches for relating LPP to memory in eCMR. After it works, we can try nonlinear features.

5 Misc

5.1 Improving Trade-Off Between Recency and Primacy

Baseline and eCMR fits seem to give low recall rates to the last few study positions in our serial position curves. One possible reason is that CMR lacks a mechanism to eliminate recency effects without boosting the strength of the primacy effect (through start-of-list context reinstatement). Since the data exhibits neither a strong recency nor a strong primacy effect, the model struggles to capture it.

On the other hand, it could be that eCMR’s complete specification is necessary to avoid this down-modulation of the serial position effect. However, since eCMR’s eMCF only affects recall rates for emotional items, I think it unlikely to help address the down-modulation observed for the neutral-SPC.

A common approach in retrieved-context models to attenuate the recency effect is to introduce a delay drift rate following the study phase, causing context to drift toward an out-of-list context before retrieval begins. The rationale is that by shifting the contextual cue away from its final study state, recently studied items will receive less contextual overlap and hence less retrieval advantage.

However, this mechanism generally fails to balance primacy and recency because it does not alter the relative similarity between the contextual cue and the study-phase contexts of individual items. Context moves equally far from all studied items, preserving their relative strengths as retrieval cues.

To overcome this limitation, I tested a modified approach where post-study context drifts not toward an external or out-of-list context, but toward all in-list contexts, where the input vector is essentially a ones mask over context units that are linked to any studied item. This adjustment removes temporal directionality from the final cue, effectively making recall initiation reflect just the differences in relative strength across context-to-item associations.

This didn’t help either, so I’m leaving this problem for later..
