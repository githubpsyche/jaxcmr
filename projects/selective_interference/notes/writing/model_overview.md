# Model Overview — Section Notes

## Structural logic

One section ("# Overview of the Model"), no subsection headings, nine paragraphs. Each mechanism is introduced alongside its paradigm application in a single pass, eliminating the redundancy that the previous two-subsection structure created.

**Why one section (vs. two):** The original structure split framework introduction (subsection 1) from paradigm application (subsection 2). But subsection 1 already applied each mechanism to the paradigm (intrusions in ¶2, reminders in ¶3, recognition in ¶4), forcing subsection 2 to restate. Merging lets each mechanism and its application appear together, and the argumentative payoff (DRA comparison, graded immunity prediction) follows naturally.

**Why explicit DRA mapping here (vs. Discussion):** The paper challenges three specific DRA assumptions. Naming the replacements in the overview lets the simulations be read as tests of predictions, not explanations after the fact.
