# Writing Constraints

## Prose constraints (apply everywhere)

| ID | Constraint | Origin |
|---|---|---|
| C11 | **One sentence, one job.** No overextension with em-dashes, semicolons, or colons to express multiple ideas. These punctuation marks can still be used if they don't violate the principle. | p1_framing v2: em-dash stapled two jobs; colon appended restatement |
| C12 | **Active voice.** Passive and stative copula constructions weaken sentences. Prefer active verbs. Functional register ("preserve," "return," "surface") naturally produces active voice; mechanistic register ("encoded," "gated," "reactivated") invites passives. | p1_framing v2: "The encoding is so strong that" (stative copula), "The result is a memory that" (weak copula). v7 confirmed: mechanistic vocabulary → passive voice |
| C15 | **Conciseness.** Target 4–5 sentences per paragraph. | p1_framing v3: 6 sentences exceeded target, lost punch |
| C16 | **Accessible register.** Avoid gratuitous jargon. Functional language is acceptable when precise. | p1_framing v4: "organism" (evolutionary biology), "retrieval governance" (technical architecture) alienated broad readership |
| C17 | **Rhetorical framing must be earned.** A sentence can only invoke a framing concept (e.g., "puzzle," "paradox," "challenge") if that concept has been established by prior sentences. Unearned framing presupposes a reader state that doesn't exist. | P3 drafting: "sharpens the puzzle" presupposed a puzzle the reader didn't yet have |

## Arc constraints (apply across paragraphs)

| ID | Constraint | Scope | Origin |
|---|---|---|---|
| A1 | **Linking phrases must connect.** Each paragraph's opening phrase must refer to a concept actually present in the prior paragraph. | intro | opening_framing: cross-paragraph coherence requirement |
| A2 | **Vocabulary consistency.** Key terms should have one meaning across the opening. If a word is used in two senses, the reader will stumble. | intro | opening_framing: "selectivity" thread (P1 functional demand → P3 technical term → P4 refers back) |
| A3 | **Forward pull.** Each paragraph should leave the reader wanting what the next paragraph delivers. | intro | opening_framing: each paragraph must motivate the next |
| A4 | **Structurally distinct from repfr.** Both papers may be read by the same reviewers. Different sentence structures and rhetorical moves. | intro | p1_framing C8: rejected beat-for-beat repfr parallel (too recognizably similar) |
| A5 | **Hedging calibrated to epistemic status.** Empirical claims hedged where findings are heterogeneous. Theoretical framing uses "in this account" / "suggest" rather than "we show." The opening frames; the paper argues. | intro | p1_framing v9: "suggest" chosen over "illustrate" — deliberate epistemic hedge |
| A6 | **Standalone self-sufficiency.** The opening must function as a complete, circulatable draft without subsections. Every concept needed to evaluate the thesis must be introduced within P1–P5. No sentence should presuppose content from a later section that doesn't yet exist. | intro | P5 bridge sentence (S2) added for this reason; P4 S5 revised to name all three DRA commitments using P5's vocabulary |

## Per-paragraph constraints

### Introduction

**P1** — Tension: durability vs. selectivity. Full derivation history in `p1_framing.md` (v1–v9).

| ID | Constraint |
|---|---|
| P1-C1 | Establish a genuine tension between two functional demands. |
| P1-C2 | Imply the theoretical perspective (context-based, single-system) without naming it. |
| P1-C6 | The tension must be genuine — the demands must pull against each other. Resolved via rhetoric (adversarial framing, friction connectives, balance metaphor), not mechanism assertion. |
| P1-C7 | Dysfunctional framing — the phenomenon of interest is what happens when one demand overwhelms the other. |
| P1-C9 | Signal trauma early. |
| P1-C10 | No implication that trauma is beneficial. |
| P1-C13 | Name the problem, not the mechanism. Relaxed for P4–P5 (which must name mechanisms at functional level). |
| P1-C14 | Traumatic memory ≠ pathological memory. The pathological case (intrusive retrieval) is the minority outcome. |

**P2** — The paradigm.

| ID | Constraint |
|---|---|
| P2-C1 | Introduce the trauma-film paradigm as a concrete instantiation of P1's tension. |
| P2-C2 | Establish the two retrieval modes (involuntary and voluntary) as a dissociation, not separate systems. |
| P2-C3 | Hedging: the dissociation is empirical ("can be weak," "can come apart"), not absolute. |

**P3** — The phenomenon.

| ID | Constraint |
|---|---|
| P3-C1 | Name the selective interference effect. |
| P3-C2 | Cover both immediate and delayed (post-reminder) evidence. |
| P3-C3 | The final sentence should sharpen the puzzle — same experience, different retrieval outcomes. |

**P4** — The dual representation account.

| ID | Constraint |
|---|---|
| P4-C1 | DRA introduced as "a prominent interpretation," not "the prevailing view." |
| P4-C2 | DRA presented as an account of selective interference, not a general primer. The paragraph answers: how does DRA explain the phenomenon P3 described? |
| P4-C3 | Two traces named at functional level (what they do — support intrusions vs deliberate recall), not encoding-process level (how they are formed). Subsection's job. |
| P4-C4 | Covers both immediate (resource competition during consolidation) and delayed (reconsolidation) mechanisms. |
| P4-C5 | Final sentence names what DRA commits to, using the same vocabulary P5 S5 will challenge (separate traces, modality-specific disruption, reconsolidation window). Sets up a clean handoff. |

**P5** — The RCT thesis.

| ID | Constraint |
|---|---|
| P5-C1 | Thesis scoped as parsimony argument ("without requiring ... as modeling prerequisites"), not blanket dismissal. |
| P5-C2 | Names all three DRA assumptions the paper challenges: separate traces, reconsolidation windows, modality-specific disruption. |
| P5-C3 | Names both mechanisms: context overlap (interference) and retrieval control (protection). |
| P5-C4 | Standalone scaffolding — one sentence characterizing the general framework before applying it. The opening circulates before subsections are written; a reader unfamiliar with RCT needs a bridge from the framework name to its application. |
| P5-C5 | Closing motivation — P5 must close with a statement of why the alternative matters beyond parsimony: theoretical unification, divergent predictions, or clinical implications. |

### Overview of the Model (single section, 9 paragraphs)

| ID | Constraint |
|---|---|
| MO-C1 | Accessible to clinical readers. No assumed familiarity with CMR, RCT, or list-learning paradigms. |
| MO-C2 | Vocabulary consistency with P5. "Items are bound to a continuously evolving temporal context" (P5 S2). "Context overlap" and "retrieval control" (P5 S3–S4). Don't introduce synonyms. |
| MO-C3 | No equations or formal specification. Conceptual only. Formal specification belongs in Model and Paradigm. |
| MO-C4 | Each mechanism named and connected to an observable consequence (what it predicts, not just what it is). |
| MO-C5 | DRA mapping must be explicit — three DRA assumptions, three CMR mechanisms, point-by-point. |
| MO-C6 | Must set up the simulations. After reading this section, the reader should understand why each simulation tests what it tests. |
| MO-C7 | Prose constraints C11–C17 apply. Arc constraints A1–A6 apply across the intro→overview boundary. |

### Model and Paradigm (5 subsections)

| ID | Constraint |
|---|---|
| MP-C1 | Equations must match `jaxcmr.models.cmr` implementation exactly. Verify against code before writing. |
| MP-C2 | Interleave formal specification with paradigm mapping. Each equation is introduced alongside its functional role in the trauma-film paradigm. |
| MP-C3 | Sculpt for simulations. Emphasize scale factors and parameters that the simulation sections manipulate. Each subsection must name which simulation(s) depend on its content. |
| MP-C4 | Self-contained. A reader unfamiliar with CMR should be able to follow the formal specification without the Overview. |
| MP-C5 | Three retrieval modes must be formally distinguished. Context→item (intrusions/free recall), item→context (recognition), and the unguided/directed operating-point distinction within context→item. |
| MP-C6 | Recognition operationalization must be explicit. The claim that recognition is architecturally immune depends on the reader understanding exactly how it is computed. |
| MP-C7 | Parameter table must include all base parameters and all scale factors, with roles and simulation connections. |
| MP-C8 | Prose constraints C11–C17 apply. Equation-heavy prose may require relaxed C15 (sentence count) where necessary for clarity. |
